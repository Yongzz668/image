Docker 运行命令

docker run -d \
  --name emby-nginx \
  -p 7001:7001 \
  -p 8021:8001 \
  -p 9527:9527 \
  -p 5002:5002 \
  -v /volume1/emby-nginx/strm:/strm \
  -v /volume1/emby-nginx/backup:/app/backup \
  --network bridge \
  --restart unless-stopped \
  --privileged \
  yantaocheng/emby-nginx:latest

Docker Compose 配置

version: '3.8'

services:
  emby-nginx:
    image: yantaocheng/emby-nginx:latest
    container_name: emby-nginx
    ports:
      - "7001:7001"
      - "8021:8001"
      - "9527:9527"
      - "5002:5002"
    volumes:
      - /volume1/emby-nginx/strm:/strm
      - /volume1/emby-nginx/backup:/app/backup
    network_mode: bridge
    restart: unless-stopped
    privileged: true

STRM生成配置教程

注意：多个emby容器要对应选对应名称的cookie模块来设置strm生成模块

添加本地STRM（给emby挂载的）

  1. 选strm生成
  2. 添加本地strm
  3. cookie模块默认别动
  4. / 意思就是整个115网盘都监控生成
    如只想生成特定文件夹就/根目录下的某个文件夹
    例如：/影视库
  5. 默认别改动，生成的strm文件都保存这个本地文件夹里
  6. 路径前缀 填写 http://本地ip:9527
    例如: http://192.168.50.178:9527
  7. 元数据下载选择，建议选否。除了字幕以外，图片和nfo没必要且一个一个下载很慢,emby自己刮削还更快
  8. 一定记得点 保存配置
  9. 立即生成strm
  10. 可以看到日志里显示生成strm

全部生成完毕strm后可以打开实时网盘监控，有新增移动删除网盘内文件都会被监控到对应生成删除strm文件

添加外网STRM（给infuse vidhub 芝杜等通过webdav挂载的）

  1. 选strm生成
  2. 添加外网strm
  3. 选cookie名称模块
  4. / 代表生成整个115网盘的strm文件，
    单个文件夹例如：/媒体文件夹
  5. 别改动 外网strm文件保存文件夹，
    可用wabdav挂载给infuse vidhub 芝杜播放机等读取播放。端口号:5002 挂载方式如vidhub看（附图1）
  6. 填你的emby-nginx外网访问地址和端口
  7. 一定记得点 保存配置
  8. 立即生成strm
  9. 日志有生成记录

infuse 芝杜 senplayer forward humhub也是这样webdav挂载的

AList配置教程

注意：设置alist 添加aliststrm模块(这是针对115网盘以外其他网盘的，115网盘用户可跳过学习)

从来没接触过alist的请看视频教程必须先学会把网盘挂载到alist

拉取安装AList命令（群晖）

docker run -d \
  --restart=always \
  -v /etc/alist:/opt/alist/data \
  -p 5244:5244 \
  -e PUID=0 \
  -e PGID=0 \
  -e UMASK=022 \
  --name="alist" \
  xhofe/alist:latest

alist的密码默认是随机生成的，要在docker容器日志里找到随机密码 登录进alist界面 然后改成自己常用的密码。

要先提前设置好alist， alist的挂载请在alist官方文档网页学习，或者通过b站视频教程去学习。

AList网盘挂载设置

  1. 点 存储
  2. 选择要挂载的网盘驱动
  3. 编辑网盘挂载路径名称，这和接下来要设置的aliststrm模块中 [alist生成路径] 填空中的名称要绝对一致
  4. alist对网盘默认缓存时间是30，可以改成1 （改完后可以最快刷新到新转存变更的文件）
  5. 默认选择 302重定向 （一定要302重定向的网盘才能反代直链播放，否则占用本地带宽流量，速度很慢）
  6. 启用签名 一定不能打开，否则会被阻断无法生成strm文件
  7. 部分网盘填 用户名和密码，部分网盘填token等 （按照alist官方文档来） 然后点 保存，点主页进去看看挂载成功了没
  8. 进设置 选择 全局
  9. 签名所有 一定不能打开，否则会被阻断无法生成strm文件
  10. 保存

AList设置部分已经完毕，所有网盘都按照alist文档教程来挂载，重要的是:两处签名 要关闭

设置AList的Cookie模块

  1. 点 cookie配置
  2. 点 添加cookie
  3. 选 alist
  4. 填写 alist内网管理地址 http://你的本地ip:5244 和 alist的登录账号密码 （如果用域名转发alist外网访问或者域名反代外网访问了 也可填外网访问的alist地址，但警告：这可能会泄露你的alist外网地址的风险）
  5. 保存配置

生成AList STRM

  1. 点 strm生成
  2. 点 添加aliststrm
  3. 选 alist
  4. alist生成路径 必须和alist挂载网盘那 填的挂载路径 绝对一致
  5. 本地保存路径 别改动
  6. alist的网盘做不到实时监控 所以要填定时规则，
    例如每小时一次的定时规则填：0 * * * *
    例如每六小时一次定时规则填: 0 */6 * * *
  7. 元数据下载项 看需要，建议选 否
  8. 一定要保存配置
  9. 其他302重定向网盘同样设置（如黄圈中的 阿里云盘）
  10. 立即生成strm（可以右上角看生成日志如 附图1+附图2 代表正在生成）

Emby 媒体服务器搭建教程

Emby Docker 运行命令

复制下面的命令到终端执行：

docker run -d \
  --name emby \
  -e TZ=Asia/Shanghai \
  -e UID=0 \
  -e GID=0 \
  -e GIDLIST=100 \
  -e HTTP_PROXY=http://192.168.1.1:7890 \
  -e HTTPS_PROXY=http://192.168.1.1:7890 \
  -e ALL_PROXY=socks5://192.168.1.1:7890 \
  -v /volume1/docker/emby/config:/config \
  -v /volume1/docker/emby-nginx/strm:/strm \
  -p 8096:8096 \
  -p 8920:8920 \
  --restart always \
  amilys/embyserver:latest

复制上面的命令到终端或者 compose，修改自己的本地路径和修改 clash 代理变量的 ip:7890，复制粘贴回车等待拉取成功，然后地址栏输入你的局域网 ip:8096 进入 emby 开始设置！

Emby Docker Compose 配置

或者使用 docker-compose.yml 文件：

version: '3.8'

services:
  emby:
    image: amilys/embyserver:latest
    container_name: emby
    restart: always
    environment:
      - TZ=Asia/Shanghai
      - UID=0
      - GID=0
      - GIDLIST=100
      - HTTP_PROXY=http://192.168.1.1:7890
      - HTTPS_PROXY=http://192.168.1.1:7890
      - ALL_PROXY=socks5://192.168.1.1:7890
    volumes:
      - /volume1/docker/emby/config:/config
      - /volume1/docker/emby-nginx/strm:/strm
    ports:
      - "8096:8096"  
      - "8920:8920"  
    networks:
      - proxy_network

networks:
  proxy_network:
    driver: bridge

挂载给播放器直链播放

首先 emby 的直链播放端口是 7001，有绑定域名的就把这个 7001 给转发映射出去。因为这个 7001 端口就是反代了 emby 原始端口 8096，用这个 7001 端口才能直链播放并且不占用自己的宽带流量。（注意 7001 端口就不要再设置反代访问了，再反代就会可能重复反代播放不了，请用端口转发出去外网播放）

网页版 emby 是无法用 7001 反代播放视频的，必须要用播放器连接 emby 播放。

打开终端播放器来测试播放一下，播个视频看一下

例如:（vidhub）

查看 vidhub 左上角网速很流畅无卡顿普遍到 20-50MB/s 和 emby-nginx 日志 有 115 成功获取直链，确定就是走的直链播放了。

后期更新版本

请安装下面这个一键升级神器，可以保留所有配置和激活信息一键升降版本。做到重新安装不求人！

docker run -d \
  --name dockercopilot \
  --restart always \
  --privileged \
  --network bridge \
  -p 12712:12712 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v /volume1/data:/data \
  -e TZ=Asia/Shanghai \
  -e DOCKER_HOST=unix:///var/run/docker.sock \
  -e secretKey=aa123456 \
  0nlylty/dockercopilot:latest

或者使用 docker-compose.yml 文件：

version: '3.8'
services:
  dockercopilot:
    image: 0nlylty/dockercopilot:latest
    container_name: dockercopilot
    restart: always
    privileged: true
    network_mode: bridge
    ports:
      - "12712:12712"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - /volume1/data:/data
    environment:
      - TZ=Asia/Shanghai
      - DOCKER_HOST=unix:///var/run/docker.sock
      - secretKey=aa123456

上述命令行里默认设置了一个管理员密码: aa123456

1. 点开你要升级的容器，
2. 选择版本号 或者 latest
3. 点 更新
可在进度那看到拉取升级进度条至绿色更新完毕
